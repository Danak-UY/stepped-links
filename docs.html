🔗 Stepped Links

Stepped Links es una extensión para Raycast que permite organizar y acceder rápidamente a enlaces mediante rutas jerárquicas con soporte para búsqueda integrada.

Imagina guardar un enlace como docs → api → components → Button y acceder a él simplemente escribiendo docs api components Button.

📦 Comandos Disponibles

🔍 Search Stepped Link Activo

Buscar y abrir enlaces mediante rutas jerárquicas

➕ Create New Step Activo

Crear nuevos pasos en la estructura jerárquica

📥 Import JSON Activo

Importar configuración desde archivo JSON

📤 Export JSON Activo

Exportar configuración a archivo JSON

🌐 Import Bookmarks Pendiente

Importar desde marcadores del navegador

🌐 Export Bookmarks Pendiente

Exportar como marcadores de navegador

🏗️ Estructura de Datos

Los enlaces se almacenan en LocalStorage bajo la clave steppedRoutes con una estructura jerárquica anidada: 

{
  "raycast": {
    "_url": "https://raycast.com",
    "_name": "Raycast",
    "_search": "https://raycast.com/search?q={{q}}",
    "docs": {
      "_url": "https://developers.raycast.com",
      "_name": "Documentation",
      "api": {
        "_url": "https://developers.raycast.com/api-reference",
        "_name": "API Reference"
      },
      "guides": {
        "_url": "https://developers.raycast.com/guides",
        "_name": "Guides"
      }
    },
    "store": {
      "_url": "https://www.raycast.com/store",
      "_name": "Store"
    }
  },
  "github": {
    "_url": "https://github.com",
    "_name": "GitHub",
    "_search": "https://github.com/search?q={{q}}"
  }
}

Convenciones de claves:

🔑 _url — URL del enlace
🏷️ _name — Nombre para mostrar
🔍 _search — URL de búsqueda
📁 [other] — Sub-rutas anidadas

🔄 Cómo Funciona

Búsqueda y Navegación

1. Usuario escribe
→
2. Parsear ruta
→
3. Buscar en datos
→
4. Renderizar

Formato de Consulta

El formato de búsqueda sigue este patrón:

{ruta-partes} {separador} {búsqueda}
        ↓          ↓          ↓
    "raycast docs api - button"
Ejemplo: Escribes raycast docs api - hooks
→ Navega a raycast > docs > api
→ Reemplaza {{q}} con hooks
→ Muestra resultados de búsqueda

🎯 Ejemplo de Uso

Supongamos que configuraste esta estructura:

{
  "work": {
    "_url": "https://work.example.com",
    "_name": "Trabajo",
    "jira": {
      "_url": "https://company.atlassian.net",
      "_name": "Jira",
      "_search": "https://company.atlassian.net/browse/{{q}}"
    },
    "confluence": {
      "_url": "https://company.atlassian.net/wiki",
      "_name": "Confluence"
    }
  }
}

Acciones posibles:

Input Acción
work Muestra "Trabajo" como enlace directo
work jira Navega a Jira, muestra sub-pasos
work jira - PROJ-123 Busca "PROJ-123" en Jira
work confluence Abre Confluence
